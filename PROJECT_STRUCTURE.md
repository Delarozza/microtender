# Структура проекта MicroTender

Этот документ описывает назначение всех файлов и папок в проекте. Используйте его для навигации и понимания архитектуры.

## 📂 Корневая директория

| Файл / Папка | Описание |
|--------------|----------|
| `package.json` | Список всех зависимостей (React, Ethers, Hardhat, Tailwind) и скрипты запуска (`npm start`, `npm test`). |
| `hardhat.config.js` | Конфигурация среды разработки Hardhat. Здесь указана версия Solidity (0.8.19) и настройки сети Polygon Amoy. |
| `tailwind.config.js` | Настройки CSS-фреймворка Tailwind (цвета, шрифты, пути к файлам для сканирования стилей). |
| `postcss.config.js` | Конфиг для обработки CSS (нужен для работы Tailwind). |
| `thesis.pdf` | Текст вашей бакалаврской работы (исходник LaTeX или скомпилированный PDF). |
| `Suhrnna_sprava.pdf` | Отчет по UX/UI тестированию и прототипам. |
| `README.md` | Краткое описание проекта и инструкция по запуску. |
| `ROADMAP-DETAILED.md` | Подробный план развития проекта и написания диплома. |

---

## 📂 Crypto inf (Блокчейн)

Здесь находится всё, что связано со смарт-контрактами.

| Файл / Папка | Описание |
|--------------|----------|
| `contracts/MicroTender.sol` | **Главный файл смарт-контракта.** Содержит всю бизнес-логику: структуры (`Tender`, `Bid`), функции (`createTender`, `vote`), события и управление ролями. |
| `.states/` | (Сгенерировано) Состояние локальной виртуальной машины Hardhat. |

---

## 📂 src (Frontend / React)

Исходный код веб-приложения.

### Корневые файлы src/
| Файл | Описание |
|------|----------|
| `index.js` | Точка входа в React-приложение. Рендерит компонент `App` в DOM. |
| `App.jsx` | **Главный компонент.** Здесь настроен роутинг (навигация между страницами) и базовая структура (Layout). |
| `index.css` | Глобальные стили и подключение директив Tailwind (`@tailwind base`...). |
| `setupTests.js` | Настройки для запуска тестов (Jest). |

### 📂 src/components (Компоненты)
| Файл | Описание |
|------|----------|
| `Header.jsx` | Верхняя панель. Содержит кнопку **Connect Wallet** (подключение MetaMask) и переключение темы. |
| `Sidebar.jsx` | Боковое меню навигации. Ссылки меняются в зависимости от роли пользователя (Admin/Member/Vendor). |

### 📂 src/components/screens (Страницы)
| Файл | Описание |
|------|----------|
| `Dashboard.jsx` | Главная панель для члена совета. Статистика, быстрые действия. |
| `AllTenders.jsx` | Список всех активных тендеров (доступно всем). |
| `MyTenders.jsx` | Список тендеров, созданных текущим пользователем. |
| `CreateTender.jsx` | **Форма создания тендера.** Здесь происходит загрузка файлов в IPFS (через `pinata.js`) и вызов `createTender` в контракте. |
| `TenderDetail.jsx` | **Детальный просмотр тендера.** Здесь отображаются предложения (Bids), происходит подача предложений и процесс голосования. |
| `VendorRegistration.jsx` | Форма регистрации для новых поставщиков. |
| `VendorApproval.jsx` | Панель для члена совета, где он одобряет или отклоняет заявки поставщиков. |
| `Voting.jsx` | (Опционально) Отдельная страница или компонент для процесса голосования (часто встроено в TenderDetail). |

### 📂 src/utils (Утилиты)
| Файл | Описание |
|------|----------|
| `pinata.js` | Логика взаимодействия с API **Pinata (IPFS)**. Функция `uploadFileToIPFS` и `uploadJSONToIPFS`. |
| `category.js` | Список категорий тендеров (константы). |
