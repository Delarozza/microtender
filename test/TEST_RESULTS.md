## Исправления в контракте

### Проблема с проверкой ролей

**Проблема:** В Solidity, неинициализированное значение enum в mapping возвращает 0 (Member), поэтому проверка `userRoles[msg.sender] >= UserRole.Member` всегда была true для всех адресов.

**Решение:** Добавлен mapping `hasRole` для отслеживания явно установленных ролей:

```solidity
mapping(address => bool) public hasRole;

modifier onlyMember() {
    require(hasRole[msg.sender] && userRoles[msg.sender] >= UserRole.Member, "Only members");
    _;
}
```

## Известные ограничения

1. **Ethers v6 vs v5:** Тесты используют ethers v6 API (`parseEther` вместо `utils.parseEther`, `waitForDeployment` вместо `deployed`)
2. **События:** В ethers v6 события доступны через `receipt.logs` и требуют парсинга через `interface.parseLog()`
3. **BigNumber:** В ethers v6 используется `BigInt` вместо `ethers.BigNumber`
